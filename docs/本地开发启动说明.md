# 本地开发启动说明

## 1. 当前可用开发栈

当前仓库已经验证可本地启动以下两部分：

- 后台后端：FastAPI
- 后台前端：`frontend/admin-web` 中的 `web-antd`
- 前台前端：`frontend/portal-web`
- 本地开发数据库：Docker PostgreSQL
- 生物信息二进制环境：Pixi（`samtools` / `bcftools` / `tabix` / `bedtools`）

对应地址：

- 后端：`http://127.0.0.1:8002`
- 后台前端：`http://127.0.0.1:5666`
- 后台登录页：`http://127.0.0.1:5666/auth/login`
- 前台前端：`http://127.0.0.1:5677`

如需快速准备公开 demo 数据，可执行：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/provision-public-demo-datasets.sh
```

该脚本会确保以下公开 demo dataset 存在并可查询：

- `demo_genome`：`sequence`
- `demo_variants`：`variant`
- `demo_expression`：`expression`
- `demo_annotation`：`annotation`
- `demo_annotation_gff`：`annotation`
- `demo_signal`：`signal`
- `demo_interaction`：`interaction`

## 2. 启动后端

首次需要先安装 Python 依赖：

```bash
cd /Users/kentnf/projects/fan-ce/backend/api-server
uv venv --python /opt/anaconda3/bin/python3.11 .venv
uv sync --python .venv/bin/python
```

当前默认开发数据库配置：

- `backend/api-server/conf/config.dev.yaml`
- 默认 `database.type = pgsql`
- 默认 `app.db_init = true`
- 默认 `app.seed_data = true`
- 默认本地开发连接：
  - `host = 127.0.0.1`
  - `port = 5433`
  - `database = fan_api_poc_dev`
  - `user = fan_api`
  - `password = fan_api_dev`
- 默认开发管理员：
  - `username = admin`
  - `password = Admin123456`

推荐先启动 Docker PostgreSQL：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/start-postgres.sh
```

对应 Docker Compose 配置：

- `docker-compose.yml`
- 初始化脚本：
  - `docker/postgres/init/001-extensions.sql`

推荐先检查 Pixi 生物信息工具环境：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/check-bio-tools.sh
```

启动命令：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/start-backend.sh
```

说明：

- 后端代码现已收敛到 `backend/api-server`
- 启动脚本会优先使用 `backend/api-server/.venv`，不存在时回退到仓库根目录 `.venv`
- 启动脚本会优先通过仓库根目录 `pixi.toml` 激活生物信息工具环境
- 使用 `--reload`
- 只监听后端相关目录
- 避免前端 `node_modules` 变更导致后端反复重载
- 默认依赖本地 Docker PostgreSQL，而不是共享 MySQL
- 若默认 PostgreSQL 在开发机不可用，启动脚本会自动回退到 `APP_ENV=dev-sqlite`
- `dev-sqlite` 只用于本地兜底启动，不改变 PostgreSQL 作为主开发方案的设计方向
- 开发环境启动时会自动建表并注入最小管理员、角色和菜单 seed
- 首次通过 `pixi` 启动时会自动解算并安装二进制依赖，时间会略长
- 如果绕过启动脚本直接用 `uvicorn` 启动，而当前 shell 没有激活 Pixi 环境，`variant / annotation / signal / interaction` 等依赖 `bcftools/tabix` 的查询会返回明确的 `503 required system binary is not available`

## 4. 启动后台前端

首次需要安装依赖：

```bash
cd /Users/kentnf/projects/fan-ce/frontend/admin-web
corepack pnpm install
```

启动命令：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/start-admin-web.sh
```

说明：

- 推荐从 `frontend/admin-web` 根工作区启动和构建，不要直接在 `apps/web-antd` 子目录单独执行 `build`
- 正确开发入口等价于：

```bash
cd /Users/kentnf/projects/fan-ce/frontend/admin-web
pnpm dev:antd
```

- 默认端口固定为 `5666`
- 当前已启用 `strictPort`，如果本机已有其它进程占用 `5666`，后台前端会直接启动失败，而不是自动切到其它端口
- 当前后台前端已提供 `Dataset 中心` 工作台，覆盖：
  - dataset 列表与详情
  - staging 上传、注册、删除
  - 生命周期主流程 `validate / index / pipeline`
  - version 工作台：创建版本、切换当前版本、查看版本详情
  - 后端版本语义已支持 `release / withdraw / set-default-public`
  - dataset 列表上的旧 `publish / unpublish` 主入口已收敛，发布动作主要在版本工作台执行
  - 公开语义已从“单 published version”升级为“多个 released versions + 一个 default public version”
  - asset / file 管理：创建、编辑、删除资产；给资产登记、编辑、删除附属文件
  - lineage 管理：创建和删除版本级、asset 级血缘关系
  - 版本工作台已展示版本发布记录，便于回看 `publish / unpublish` 轨迹
  - 版本工作台已支持版本级 query tester，可直接加载 capabilities、填充示例参数并执行查询
  - 类型注册表管理：超级管理员可维护 `dataset kind` 与 `asset type`
- 当前后台 dataset API 已补第一阶段数据权限：
  - 菜单权限之外，再按 dataset owner、team、project 做行级控制
  - version / asset / asset_file / lineage / ingest task 默认继承 dataset 权限
  - 超级管理员和系统管理员可全量访问

## 5. 启动前台前端

首次需要安装依赖：

```bash
cd /Users/kentnf/projects/fan-ce/frontend/portal-web
corepack pnpm install
```

启动命令：

```bash
cd /Users/kentnf/projects/fan-ce
scripts/dev/start-portal-web.sh
```

说明：

- 默认端口是 `5677`
- 本地开发代理到 `http://127.0.0.1:8002/api/v1`
- 当前前台已提供公开数据门户页面，覆盖：
  - 公开 dataset 目录浏览
  - 公开 dataset 详情查看
  - released public versions 切换，默认落在 `default_public_version`
  - 基于公开 `query capabilities` 的在线查询面板
  - 根据 adapter/capabilities 驱动 operation 选择与查询入口展示
  - `sequence` 适配器下的结构化查询表单：`fetch(seq_id/start/end)` 与 `batch_fetch(regions)`
  - `variant` / `expression` / `annotation` / `signal` / `interaction` 适配器下的 smart 表单骨架，仍保留 JSON 模式作为兜底
  - 原始 JSON 查询模式，便于兼容其它适配器或调试请求
- 当前公开前台暂不展示 lineage，后续如需开放，应单独评估公开版本与私有依赖关系的暴露边界
- 公开查询示例参数会根据实际 FASTA `.fai` / header 动态生成，避免示例 `seq_id` 与真实数据不一致

## 8. 当前已知问题与说明

- 后端启动时有 `passlib/bcrypt` 警告，但不影响当前登录和接口访问
- 前端存在 `Directive "loading" has already been registered` 警告，但不影响当前开发运行
- 部分菜单页面目前是补齐的占位页，后续可替换成正式功能页面
- 如果本机 Docker / PostgreSQL 不可用，后端会自动回退到 `dev-sqlite` 继续启动
- 如果本机 `5666` 已被占用，后台前端会因 `strictPort` 直接启动失败，需先释放该端口
- 如果本机 `5677` 已被占用，前台前端需调整 Vite 端口
- 真实 `index` 能力依赖 `samtools/bcftools`；现在已改为由 Pixi 管理，不再依赖手工装到系统 PATH
- 真实 `variant / annotation / signal / interaction` 查询也依赖 `bcftools/tabix`；建议统一通过 `scripts/dev/start-backend.sh` 或等价的 Pixi 环境启动后端
